2. 論点とは何か
実践

論点は一文で明文化する

「べき」の入った論点を明確にすることは簡単なことではない。

特にちょっとした問題であれば論点を曖昧にしたまま行動に移ってしまうことが多い。

日常生活であればそれで問題ないかもしれないが、ビジネスシーンであればこれは極めて非効率な行為である。論点が明文化されていないままぼんやりと考えて行動を起こすと結局のところ本当に欲しい答えに辿り着くのに不必要に多くの時間が費やされてしまったり、あるいは間違った結論に辿り着き行動が無駄になってしまったりすることが往々にしてある。どんなに細かいことであっても僅かでも迷いが生じたならば、自分がどのような論点を解こうとしているのかを明文化するべきなのである。

3. 論点を分解する
How

イシューツリーは「真ん中あたり」からつくるべきである

結局のところイシューツリーはスパッとトップダウンで大論点から中論点、小論点という順番で作ることはまずできない。そのような人がいたら余程の天才か論点が簡単かのどちらかだろう。通常は小論点を考えてその上位にある論点を考えたり、逆に大論点を考えてそれを分解してみたりしながら上位階層と下位階層を行ったり来たりしながら考えるというのが現実である。

トップダウン的に大論点を分解すると多くの場合は色彩が失われてしまい論理的には正しいけれども一般論であり個別論になっていないという事態に陥りがちである。このような状況になってしまうのは本来はプロジェクトに固有の論点が存在しているが、一般論はなまじ論理的に正しいためにそれを用いてMECEにイシューツリーを作ると固有の論点が埋もれてしまうためである。

3. 論点を分解する
How

トップダウンとボトムアップの両者で整合性を取る

トップダウン的な手法だけでなくボトムアップ的なアプローチも併せて採るべきである。大論点に答えを出すためには、検証が必要な小論点を構造などにこだわらずに思いつくままに列挙してみるのである。この際には関係者が特に気にしていた点を論点に転換してみると無味乾燥とした一般論ではなく個別の論点が浮かび上がってくる。

しかしこれでは構造化されていないためにモレがあったりダブりがあったりあるいはレベル感が揃っていなかったりする。

そのためにボトムアップで思いついた個別論点が出尽くした段階で今度はそれらを構造化してツリーに落とし込んでいくプロセスが生じる。この際にはトップダウンで作った無味乾燥としたイシューツリーとも照らし合わせ両者の整合性を取っていくことになる。

このようにイシューツリーを策定するときは教科書的にはトップダウン的なアプローチを取るが現実にはボトムアップとトップダウンの両方を行き来しながら作り込むことになる。

4. 仮説を構築する
前提

仮説とはベストエフォートの答えであり、
必ずつくることができる

仮説とは文字通り「仮の答え」であり、言い換えるとその時点でのベストエフォートでの答えである

ベストエフォートということは、その時点に持ち合わせている情報だけを用いて足りない情報は仮定を置いて構築された論理体系であるということである。

これは見方を変えるというと仮説を作れないということはあり得ないのである。

持っている情報がゼロであったとしても分からないことには全て前提をおけば一つの論理体系はできるはずであり、それをできないというのは能力の問題ではなく意思の問題であり、厳しい表現を使えば知的に怠惰であるともいえる。

絶対覚えて帰って!

4. 仮説を構築する
前提

仮の答えを出すことから逃げない

人間にとって何かを決めるということは脳にとって非常に負荷のかかる行為であり無意識のうちに決断することから逃げる傾向があると自覚する

特に今すぐ仮の答えを出す必要がない場合は、後で判断しようと思ってしまうし、またそもそも情報がないことを言い訳に調べてから答えを出そうと考える人も多い

その場で仮の答えを出さずにあとで決めようとするともう一度、論点を思い出したりする必要があるしまたそもそもその論点を検証するということを忘れてしまうリスクも存在する

何らかの論点がでたらその場で仮の答えを出して指示を出す規律を持つべきである

4. 仮説を構築する
仮説思考の罠

一方で難易度の高い論点への仮説は
一定の知見が必要になることを知る

それなりに難易度の高い論点の仮説を構築するときにこの論点に関連した知見を持っている人物がいない場合には仮説が極めて浅いものになってしまう。このような状況では論点に対して表層的な仮説を構築しそれを検証したところで説得力が全くない結論しか出てこないリスクがある。

そのため仮説を構築するときには、そもそも自分自身あるいは一緒に働いている人にこの論点に関する知見があるかを客観的に認識するべきである。客観的に、とは現実と願望を区別して、とも言い換えられる。もしもこの問いに対する答えがノーなのであれば稚拙な仮説の構築に時間を投入するよりも「筋のいい仮説を構築するためには何をするべきか？」という論点を立てるべきである。
