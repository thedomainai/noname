# AI経営変革と株主価値 — きらぼしFG Fund OS提案

ChatGPT との議論（2026-03-23）を論点別に再構成したもの。

## 1. 米国企業のAI経営変革 — 4類型

株主価値に効くAI/AIエージェント活用は4つの型に分類できる。

### 1-1. AIを新収益レイヤーにする型

AI/agent自体をARR・ARPUの源泉にする。

**Salesforce**: FY26売上 $41.5B、営業CF $15.0B、Agentforce ARR $800M（前年比+169%）、案件29,000件、$50Bの自社株買い枠。AIは「効率化ツール」ではなく成長商品であり資本還元余力の源泉。

**Intuit**: 「AIエージェント＋人間専門家」を商品体験に埋め込む型。FY25は売上+16%、GAAP営業利益+36%、EPS+31%。AIの寄与分を厳密に分解できないが、経営の中核にあり売上・利益・EPSの改善が伴っている。

### 1-2. 社内オペレーティングシステムを書き換える型

顧客対応、開発、生産性、リスク管理を内部から圧縮する。

**JPMorgan**: 20万人超にLLM Suiteを開放。production AI use caseが1年で倍増。重点はコールセンター効率、顧客インサイト、ソフトウェア開発。$600Mの効率化を特定（一部AI関連）。

**Bank of America**: Erica for Employeesは従業員の90%以上が利用し、ITサービスデスク問い合わせを50%以上削減。AskGPSは3,200超の内部文書で学習済みで、1時間かかっていた複雑問い合わせをほぼ即時処理。年間数万時間規模の削減余地。人員削減ではなく高単価人材の時間配分最適化。

### 1-3. 現場の意思決定回転数を上げる型

**Walmart**: Wallyが商品分析、過不足在庫の原因特定、運用問い合わせ対応を自社データ上で自動化。夜間補充シフト計画が90分→30分に短縮。アパレルのTrend-to-Productは工程を最大18週間短縮。Walmart自身が「agentsは汎用より、特定タスクを小さく分けて組み合わせる方が効く」と発言。

### 1-4. 経営管理そのものをリアルタイム化する型

実例より設計思想が先行。McKinseyやBainが agentic budgeting、end-to-end workflow redesign をここに位置付ける。

### 4類型の比較

| 観点 | Salesforce/Intuit型 | JPMorgan/BofA/Walmart型 |
|---|---|---|
| 株主価値への伝達速度 | 短期で直結（ARR、売上、営業利益、買戻し） | 業務KPIに先に出て、EPS/ROICに時差あり |
| 持続的競争優位 | モデル自体は模倣可能 | 自社固有データ×業務文脈×実行権限がモート |

## 2. 楽観論と悲観論の統合

### 楽観側

- **Microsoft**: humans + agents の「Frontier Firm」を提示。戦略を再考する転換点と見るリーダー82%、12-18ヶ月でagentsを組み込む81%
- **McKinsey**: 価値の源泉はagentではなくworkflow redesign。AI-first workflow、agentic budgeting、embedded governance
- **BCG**: 価値創出企業はAI収益を再投資。2028年にはagentsがAI価値の29%

### 悲観側

- **Gartner**: 2027年末までにagentic AI projectの40%以上が中止（コスト高、価値不明、リスク管理不足）。"agent washing"を指摘、本当にagenticなベンダーは約130社。ただし2028年までに日々の業務判断の15%が自律化、企業ソフトの33%にagentic AIが組み込まれるとも予測
- **Bain**: ソフトウェア開発で10-15%生産性改善があっても、プロセスを変えず浮いた時間を高付加価値業務に振り替えなければ正のROIにならない

### 結論

楽観論と悲観論は矛盾していない。両者は実行品質の違う世界を見ている。

| 成功する会社 | 失敗する会社 |
|---|---|
| workflow-first | chatbot-first |
| domain-specific | 横展開先行 |
| governed | 責任分界曖昧 |
| cash-linked | ROI不明 |

## 3. AI経営変革の理想形 — 5層アーキテクチャ

理想形は「全社Copilot配布企業」ではなく「agentic operating systemを持つ資本配分企業」。AIはIT部門の新機能ではなく、CEO/CFOの経営制御ループを短縮する装置。

| 層 | 名称 | 内容 |
|---|---|---|
| L1 | データ基盤 | 取引・顧客・在庫・コード・財務・規制データの統合。価値の源泉は汎用モデルではなくここ |
| L2 | 専門agent群 | pricing, service, merchandising, underwriting, collections, coding, forecasting等。狭く反復的な職務ごとに分割 |
| L3 | Workflow orchestrator | agent単体ではなくend-to-endで業務をつなぐ。ここを変えないとROIが出ない |
| L4 | Human-above-the-loop | 人間は実行者から目標設定・例外処理・トレードオフ判断・顧客接点に移行 |
| L5 | Embedded governance | critic agents、compliance agents、監査証跡、閾値管理、kill switch |

### 最初に狙う価値プール

短いフィードバックループを持つ高頻度・高額な判断を優先する。価格最適化、営業/CS、在庫、与信、回収、ソフトウェア開発、FP&A。理由はcash impactが測りやすく学習ループも速いから。

### 価値の定義

工数削減は価値とみなさない。以下のどれかが数字に出た時だけ価値と認める。

- 固定費の実減
- 同じ人員での処理量増
- 価格実現率改善
- churn/fraud/lossの低下
- 在庫・運転資本の圧縮

理想形は、節約した工数をさらに成長とモート形成へ再投資する自己強化ループ。

### 反証条件

- AI関連支出増にもかかわらず2-3年でFCF・ROIC・単位当たり生産性が改善しない
- 「時間削減」は出るが固定費・処理量・価格・運転資本に転写されない
- 例外処理や手戻りが増え、見えない人手が裏で膨張する
- 顧客満足・品質・コンプライアンス・モデルリスクが悪化する
- 汎用ベンダー依存が強く自社データ/業務文脈のモートが形成されない
- 競合も同じAIを使い価値が価格競争で相殺される

### きらぼしFGの現状

|---|---|

| ROE | 8.5%（3Q累計9.45%） |

| 親会社株主純利益（3Q累計） | 276.8億円 |

将来像（FY2033）: PBR 1倍以上、ROE 10%以上、資本比率 10%以上

### きらぼし版の理想形

「支店に汎用Copilotを配ること」ではなく、「法人RM起点で、融資・モニタリング・コンサル/リース/証券/キャピタル送客・資本配分を一本のループにしたグループOS」。

推論ルール: 「独自データ × 判断頻度 × 資本制約 × 統制可能性」の積が最大の所からAIを入れる。

### 5層構造（きらぼし版）

| 層 | 内容 |
|---|---|
| L1 顧客グラフ | 口座取引・入出金・与信・担保・CRMメモ・来店/通話履歴・UI銀行/アプリ行動・証券/リース/コンサル/キャピタルの接点を統合 |
| L2 専門agent | RM agent、稟議ドラフト agent、期中モニタリング agent、事業承継/資本政策 agent、AML/KYC agent、PB/相続 agent |
| L3 ワークフロー・オーケストレータ | 業務のend-to-end接続 |
| L4 人の承認 | Human-in-the-loop |
| L5 資本配分コックピット | CFO/経営会議向け。Business value = Profit - (Allocated risk capital × Cost of shareholders' equity) |

### 優先ワークフロー

**第1優先: 法人RM → 融資 → 期中モニタリング → グループ送客**

- 入力: 入出金、給与振込、税公金、借入返済、担保、過去稟議、面談メモ、業種データ、相談履歴
- 出力: 資金需要/悪化兆候の仮説、RM向け面談メモ、価格レンジ、必要書類リスト、稟議ドラフト、早期警戒、グループ送客提案
- KPI: RM1人当たり純収益、貸出利回り、手数料収益、信用コスト、紹介成約率、RORA

**第2優先: 建設業向け埋込金融**

きらぼしはMCデータプラスと共同で「建設工事サポートローン」→「建設工事サポートローン・プラス」まで進めている。工事案件データ・支払期日・実行進捗・安全書類・口座入出金をつなぎ、案件単位の資金需要推定→必要書類収集→事前スクリーニング→人間審査→実行後モニタリングを閉ループ化。

**第3優先: 事業承継・M&A・資本政策オーケストレータ**

オーナー企業の株主構成・資金繰り・借入余力・担保・家族/相続イベント・資産管理・買手候補との接点を一枚にまとめ、銀行・コンサル・キャピタル・証券へ最適に流す。RWAを増やさず手数料・エクイティ収益を増やせる。

**第4優先（第2波）: 個人/富裕層**

価値が小さいからではなく、顧客向けAIの統制がまだ重いため後回し。まず行員/アドバイザー向けnext-best-actionと面談準備assistantとして始める。

### 統制設計

各agentに必須の統制要素: リスク格付け、モデル台帳、プロンプト/出力ログ、参照根拠、override理由、定期再検証、ベンダー管理、kill switch。融資承認や適合性判断は初期段階では自動化しない。

運用原理: 金融庁「リスクベース」+「アジャイル・ガバナンス」。低リスクの社内効率化は現場判断、高リスクの顧客向けAIは専門部署が事前審査し導入後もライフサイクル全体で見直す。3線防御（一線: 継続モニタリング、二線: 独立検証、三線: 内部監査）。

### 株主価値への接続

AI案件の採否基準:
- 利回り・手数料で利益が増えたか
- RWA/配賦リスク資本当たりの収益が上がったか
- 信用コストや隠れ事務が減ったか
- 収益の安定性が増し資本コスト低下の物語を作れたか

### きらぼし版の反証条件

- 審査時間短縮のみで貸出利回り・手数料・信用コスト・business valueが改善しない
- 行員のoverrideが多く隠れ手作業が増える
- 顧客苦情、AML見逃し、利益相反問題が増加
- グループ送客が増えても成約せず、資本コスト低下の物語が市場に伝わらない

## 6. きらぼし向け Fund OS 提案

### 提案の背景

- 提案者はSBI内300億円のクリプトファンド（FoF）を一人で運用。DD〜デプロイ〜組み入れ更新〜ニュース対応・脅威度判定まで全面AI化済み
- きらぼしFGとSBIHDは2023年に業務提携（デジタルサービスの競争優位性向上）
- きらぼしのファンド業務は中核: PEファンド収益 FY2023 33億円→FY2026 86億円計画。投資先ファンド119本・投資上限1,221億円、きらぼしキャピタル運営8本・投資上限652億円

### 提案コンセプト

「きらぼし全体のファンド関連業務を回す Fund OS」。売る単位はOS、導入単位は「ファンド関連業務だけ」に絞る。

### Fund OS の3モジュール

**Module 1: Origination / DD OS**

ティーザー、CIM、財務資料、面談メモ、過去ICメモ、ニュース、公開開示、社内案件メモを入力し、初期スクリーニング・レッドフラグ・論点整理・ストラクチャ案・ICメモ草案を出力。きらぼし銀行・きらぼしキャピタル・A&KC・きらぼしコンサルティングの接点になる。

**Module 2: Monitoring / Threat OS**

組入先・候補先について、ニュース・訴訟/不祥事・業績変化・資金繰り悪化シグナル・業界ショック・キーマン異動・ガバナンス劣化をスコアリング。月次・四半期モニタリング資料を自動更新。提案者のニュース対応・脅威度判定の実運用ノウハウが最も刺さる部分。

**Module 3: Reporting / Capital OS**

各ファンド・案件・担当・戦略ごとに、コミット・組入・評価・EXIT見込み・収益寄与・未実現/実現損益・案件パイプライン・レビュー負荷を束ね、経営会議・CFO・リスク管理向け報告パックを生成。

### 提案先

渡邊壽信氏（FG代表取締役社長・グループCEO、兼きらぼし銀行頭取）。FGコーポレートプランニング / きらぼし銀行法人 / きらぼしキャピタル / A&KCの横断スポンサーが理想。

### 90秒ピッチ

> きらぼし全体のファンド関連業務を回す Fund OS の提案です。投資判断をAIに置き換える話ではなく、案件発掘・DD・投資委員会メモ・モニタリング・ニュース脅威判定・EXIT準備・経営報告を一枚の制御盤で回す話です。まずは3か月、きらぼしキャピタル・A&KC・銀行本体のファンド関連業務だけに絞って影運転します。成功指標は、ICメモ作成時間・モニタリング工数・ニュース見逃し・報告リードタイム、そして案件数増に対して人員を比例増させずに回せるか。うまくいけば事業承継・LBO・メザニン・外部ファンドLP管理まで、きらぼしの資本ソリューション全体のOSになります。

### コミュニケーション戦略

**言うべきこと:**
- 人の投資判断は残す
- 投資委員会と監査に耐える
- 案件数が増えても人員が比例増しない
- ファンド収益の取りこぼしと見逃しリスクを減らす

**言わない方がいいこと:**
- 暗号資産（相手が食いついてから出す）
- 自動運用
- PoC
- 工数削減

**自己紹介の切り口:** 「AI導入コンサルだが、PoC屋ではなく、実運用のFund workflowをAIで回している側」

**統制を最初から入れる:** human-in-the-loop、参照根拠、承認ログ、権限管理、モデル台帳、vendor管理、kill switchを提案書に最初から記載。金融庁2026年AIディスカッションペーパー、モデル・リスク管理2024プログレスレポートの文脈に乗せる。

### 着地

「3か月の影運転のスポンサーをください」
