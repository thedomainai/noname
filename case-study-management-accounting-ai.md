# 経営管理 × AI — 売上30億・3事業部体制での実践記録

## この会社について

売上約30億円、3事業部体制。経営管理機能（経営企画寄り）として、以下の業務を担当している。

- 全社・事業部別のKPI策定と運用
- 年次予算策定と期中見直し
- エクイティストーリーの構築・更新
- 各事業部との月次定例の設計・運営

経営管理の実働は少人数。やるべきことは明確だが、手が回らないものが常に積み残されていた。

## 「やった方がいいが、やれていなかったこと」の一覧

以下は、経営管理として当然やるべきだとわかっていながら、手が回らずに放置されていた業務である。高度な分析の話ではない。**どれも経営管理の基本動作**であり、できていないこと自体が問題だった。

| やれていなかったこと | 実態 |
|---|---|
| 月次の数字を月初5営業日以内に揃えること | 各事業部がバラバラのExcelで報告。統合・突合に時間がかかり、経営会議に間に合わない月がある |
| 予実差異に「なぜ」をつけること | 「売上未達 △800万」は出せるが、その原因（案件ズレ？単価下落？件数減？）まで毎月掘る余力がない |
| 予算の前提条件を文書で残すこと | 作った本人の頭の中にしかない。半年後に「この数字の根拠は？」と聞かれて誰も答えられない |
| 会議で決まったことを翌月に追跡すること | 議事録は書く。が、前回のアクションアイテムを次回冒頭で確認する習慣がなく、決めっぱなし |
| 3事業部の報告フォーマットを統一すること | 各部が独自フォーマットで出してくる。毎月手作業で統一形式に転記している |
| 月次着地見通しを出すこと | 年初予算は作るが、期中に「今期どこに着地しそうか」を月次で更新する運用がない。聞かれてから慌てて計算する |
| 取締役会資料を効率的に作ること | 毎回、前回資料をコピーして数字を差し替え、体裁を直す作業。構造は同じなのに毎回2日かかる |
| 事業部長に会議前に論点を共有すること | 会議の場で初めて数字を見せる。事業部長も準備なしで臨むため、その場で答えられない質問が飛ぶ |
| 競合の動きを定期的に整理すること | IR直前に慌てて調べる。普段は誰も追っていない |

これらは「怠慢」ではない。**少人数の経営管理チームが、目の前の締め切りを優先した結果、構造的に後回しになっていた**ものである。どの会社でも起きている。

## 何を、どう変えたか

### 1. KPI策定: 「財務KPIだけ」→「先行指標を含む多層KPI」

**Before**

KPIは各事業部の売上・粗利・営業利益の3指標が中心。月次で実績を集計し、予算比・前年比を出す。それ以上の分析（なぜ売上が落ちたか、どの先行指標が悪化したか）は、問題が顕在化してから個別にExcelで掘る作業だった。

先行指標（パイプライン件数、商談進捗率、顧客継続率等）は「追いたいが、データ収集コストが高すぎる」として後回しにされていた。

**After**

AIに以下を任せた:

- 各事業部のデータソース（SFA、会計ソフト、スプレッドシート）から月次データを収集・統合する定型プロセスの設計
- 財務KPIと先行指標の相関分析（どの先行指標が3ヶ月後の売上と相関するか）
- 異常値検知（前月比で有意に変動した指標のフラグ立て）

結果として、**追跡KPIが事業部あたり3指標→8〜10指標に拡張**された。増えたのは主に先行指標で、これにより「問題が顕在化する前に兆候を検知する」体制に移行しつつある。

データ収集の統合設計に約2週間、相関分析のプロンプト設計に1週間。以降の月次運用は、AIが下書きしたレポートを人間がレビュー・修正する形で、**月次2〜3時間で回っている**。

**変わったこと**: 経営会議で「なぜ？」の議論が始まる前に、先行指標の変動として兆候が見えるようになった。事後対処から事前検知への転換。

### 2. 予算策定: 「年1回の大作業」→「月次ローリングフォーキャスト」

**Before**

年次予算は10月から12月にかけて約3ヶ月かけて策定。各事業部からボトムアップで数字を集め、全社で調整し、取締役会で承認。期中の見直しは半期に1回が限界。

問題は、**予算が3ヶ月で陳腐化する**こと。市場環境が変わっても予算の前提は固定されたまま。「予算比」の数字が経営判断の参考にならない状態が常態化していた。

ローリングフォーキャストの導入は何度も検討されたが、「月次で3事業部のフォーキャストを更新する工数」を考えると現実的でなかった。

**After**

AIを使って以下の仕組みを構築した:

- 月次実績が確定した時点で、AIが残月のフォーキャストを自動更新（直近トレンド＋季節性＋既知のイベントを考慮）
- 人間はAIが出した着地見通しに対して「これは違う」という部分だけ修正する（例: 大型案件のクロージング時期のズレ、価格改定の影響等）
- 予算との乖離が一定幅を超えた項目は自動でフラグが立ち、コメント入力を求める

**月次フォーキャスト更新にかかる時間が、ゼロベースで作る場合の推定3日→半日に短縮**された。AIが80%の下書きを作り、人間が20%の判断を入れる構造。

**変わったこと**: 「年次予算 vs 実績」ではなく「最新フォーキャスト vs 実績」で議論できるようになった。年度末に「予算未達だが年初の前提が間違っていた」という不毛な議論がなくなる。

### 3. エクイティストーリー: 「資金調達時に急いで作る」→「常時アップデート」

**Before**

エクイティストーリーは資金調達ラウンドの直前に集中的に作成していた。市場データの収集、競合分析、自社の成長ストーリーの整合性チェック、財務モデルの更新——これらを2〜3週間で一気にやる。

問題は2つ:
- 調達が終わると更新が止まり、半年後には現実と乖離する
- 急いで作るため、市場データが浅い（1〜2日で集められる範囲に限定される）

**After**

AIに以下を継続的に実行させている:

- 競合の公開情報（プレスリリース、決算情報、採用動向）を月次で収集・要約
- 市場規模・成長率の推計に使う公開データの定点観測
- 自社KPI実績をエクイティストーリーの前提と突合し、乖離が出た箇所をフラグ

エクイティストーリー本体の書き換えは人間が行うが、**「書き換えるべき箇所」の特定と、書き換えに必要な材料の収集**がAIで自動化された。

**変わったこと**: 投資家や取締役から突然「最新の市場環境を踏まえた事業計画の見通しは？」と聞かれても、1〜2日で更新版を出せる状態になった。以前は2〜3週間かかっていた。常に「半完成品」が手元にある状態。

### 4. 月次定例: 「報告会」→「意思決定会議」

**Before**

月次定例の準備に毎月3〜4日かかっていた。内訳:

| 作業 | 所要時間 |
|---|---|
| 各事業部からの数字収集・突合 | 1日 |
| 全社サマリ資料の作成 | 1日 |
| 前月比・予算比の差異分析コメント作成 | 0.5〜1日 |
| 資料のフォーマット調整・体裁整え | 0.5日 |

会議自体は2時間。そのうち**1時間30分が「何が起きたか」の報告**、残り30分が「どうするか」の議論。報告に時間を使いすぎて、意思決定の時間が常に不足していた。

さらに、前回会議で決まったアクションアイテムが追跡されないまま次の会議を迎えることが常態化していた。事業部長には会議の場で初めて数字を見せるため、その場で答えられない質問が飛び、「持ち帰り」が増える悪循環だった。

**After**

AIを使って、会議の準備から事後フォローまでの一連のプロセスを再設計した:

- 数字の集計・突合・前月比/予算比の算出を自動化し、3事業部の報告を統一フォーマットで出力
- 差異の大きい項目に対するコメント下書き（「売上が前月比-8%。主因はBU-2のX案件の失注（△2,000万円）と推定」）
- 前回定例のアクションアイテム一覧と進捗状況を自動整理し、未完了項目をフラグ
- 事業部別の論点サマリを生成し、会議2日前に事業部長へ事前配布（「今月の主要論点: BU-1は粗利率改善の施策効果検証、BU-3は新規案件パイプラインの質」）

人間が担うのは、AIの分析の妥当性チェック（数字の裏にある文脈はAIにはわからない）、論点の優先順位づけ（「今月はBU-2の失注分析を重点的に議論する」）、会議のファシリテーションである。

**準備時間が3〜4日→1〜1.5日に短縮**された。レポートテンプレートの設計に約1週間、最初の2ヶ月はAI出力を全項目チェックして精度を上げた。現在はレビュー中心で回っている。

**変わったこと**: 会議の構造が根本から変わった。事前に論点付きレポートが配布されるため、会議冒頭の報告が15分で済み、**残り1時間45分が意思決定と議論に充てられる**。同じ2時間でも、中身が報告から意思決定に変わった。

## 投入リソースと期間

| フェーズ | 内容 | 期間 | 工数 |
|---|---|---|---|
| 設計 | 業務棚卸し、AIで代替可能な範囲の特定、データフロー設計 | 2週間 | 経営管理担当1名のフルタイム相当 |
| 構築 | プロンプト設計、データ連携、テンプレート作成 | 3〜4週間 | 同上 |
| 並走 | AIの出力を人間がチェックしながら精度を上げる | 2ヶ月 | 月20時間程度（通常業務と並行） |
| 定常運用 | AIが下書き → 人間がレビュー・修正 | 継続中 | 月15〜20時間（従来比 約40%減） |

初期の設計〜構築フェーズで集中的に時間を使うが、外部ツールの大規模導入やシステム開発は不要。既存のデータソース（会計ソフト、SFA、スプレッドシート）とAIの組み合わせで成立する。

## 本質的に何が変わったか

工数が減ったこと自体は副次的な成果である。本質的な変化は5つある。

### 1. 経営者の会議での時間の使い方が変わった

以前の月次定例は、最初の1時間半が「何が起きたか」の報告だった。事業部長が数字を説明し、経営者が質問し、その場で原因を推測する。議論が白熱するのは大抵「なぜこの数字なのか」という事実確認であって、「だからどうするか」という意思決定ではなかった。

今は事前に論点付きレポートが配布されるため、会議が始まる時点で全員が数字と主要な差異の原因を把握している。会議冒頭の報告は15分で済み、残りの時間が「BU-2の失注が続いている原因は営業体制なのか、プロダクトなのか」「BU-3のパイプラインが薄いが、来期の採用計画を前倒しするか」といった意思決定に使われるようになった。

経営者にとって一番貴重なのは時間である。同じ2時間の会議でも、中身が報告から意思決定に変わることで、**会議1回あたりに下せる判断の数が体感で2〜3倍になった**。

### 2. 問題の検知が「事後」から「兆候」に変わった

以前は、売上が落ちたことを月次の数字で知る。知った時点で、その月はもう終わっている。原因を掘り、対策を決め、実行するのは翌月以降。問題の発生から対処開始まで最短でも1ヶ月のタイムラグがあった。

今は、月次の着地見通しが常に更新されているため、月の半ばで「今月の着地が予算を下回りそうだ」という兆候が見える。BU長に「下振れ要因は何か、リカバリーの手はあるか」と月内に確認できる。

これは「速くなった」のではない。**以前は構造的に不可能だったこと**が可能になった。月次サイクルでしか回していなかった管理を、事実上週次に近い粒度に引き上げられた。

差は小さく見えるかもしれないが、年間で考えると大きい。問題を1ヶ月早く検知して対処を始められるなら、年間12回の月次サイクルで**12ヶ月分の対処リードタイムが生まれる**。

### 3. 経営管理の仕事が「集計」から「解釈」に変わった

以前の経営管理担当の1週間を振り返ると、工数の7割はデータを集める・揃える・転記する・フォーマットを整える作業だった。残りの3割で分析や資料の中身を考える。しかし3割では深い分析はできず、結局「数字を並べて簡単なコメントを付ける」レベルで精一杯だった。

今はデータ収集・集計・フォーマット整備をAIが担うことで、**人間の時間の7割が「この数字をどう解釈するか」「経営者に何を提言するか」に使える**ようになった。

具体的に変わった仕事の例:
- 以前: 3事業部のExcelを1つのフォーマットに転記する作業に半日
- 今: 転記はAIが済ませた状態で、「BU-1の粗利率が3ヶ月連続で下がっている。原因は原価の上昇か、値引きの増加か、ミックスの変化か」を考える時間に半日を使う

やっていることの難易度が違う。転記は誰でもできるが、粗利率低下の構造分析は経営管理の専門性が求められる。**AIが単純作業を引き受けたことで、人間が専門性を発揮する時間が生まれた**。

### 4. 組織の記憶が「個人の頭」から「文書」に移った

以前は、多くの重要情報が特定の人の記憶に依存していた。

- 予算の前提条件 → 作った人しか知らない
- 先月の会議で何が決まったか → 議事録はあるが、探すのに時間がかかる
- あの数字の根拠は何か → 「確かExcelのどこかに…」

人が異動したり退職したりすると、この記憶は失われる。新任者は「なぜこの予算なのか」の背景がわからないまま、前任の数字を引き継ぐことになる。

AIを使ったプロセスでは、予算の前提条件、差異分析の理由、会議のアクションアイテムと進捗がすべて構造的に記録される。AIがレポートを生成する過程で、**記録すること自体が業務フローに組み込まれた**ため、「記録する手間」が追加コストとして発生しない。

半年前の取締役会で「来期は新規事業に投資する方針」と決まっていたのに、予算にそれが反映されていない——こうした「決めたのに忘れた」問題が構造的に起きにくくなった。

### 5. 当たり前のことが当たり前にできるようになった

冒頭の「やれていなかったこと」の一覧を見返してほしい。どれも経営管理の基本動作である。予実差異に「なぜ」をつける、会議で決まったことを追跡する、予算の前提を文書に残す——特別なことは何もない。

やっていなかったのは、知らなかったからではなく、**やるコストが人手に見合わなかったから**だ。少人数のチームで月次の締めを回すだけで手一杯で、基本動作に充てる余力が残らなかった。

AIがこの状況を変えた。単純作業のコストが下がったことで、基本動作に手が回るようになった。

この変化は地味に見える。しかし実際には、**当たり前のことを当たり前にやっている会社は驚くほど少ない**。月次の数字が月初5営業日以内に揃い、予実差異に毎月理由がつき、会議の決定事項が翌月に追跡され、予算の前提が文書として残っている会社。これだけのことがすべてできている30億円規模の会社は、体感では2割もない。

だからこそ、基本動作の徹底はそれ自体が競争力になる。凡事徹底を可能にした、というのがAI導入の最も本質的な成果だと考えている。
