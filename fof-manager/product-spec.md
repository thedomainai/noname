# fof-manager プロダクトドキュメント

## プロダクトコンセプト

### ターゲットユーザー

**プライマリ**: シニアファンドマネージャー（FoF運用責任者）
- 役割: ポートフォリオ全体の運用責任者
- 主要タスク: 月次・四半期でのポートフォリオレビュー、ファンド解約・新規契約の意思決定
- ゴール: ポートフォリオ全体を理想プロファイルに近づける、市場ショック時の原資産減少を最小化

**セカンダリ**: アナリスト（ファンドDD・モニタリング担当）
- 役割: 個別ファンドのDD・モニタリング担当
- 主要タスク: ファンド情報の更新・定量指標の計算・定性情報の収集
- ゴール: 最新情報を正確に記録し、異常値・リスク兆候を早期検知

**ターシャリ**: CIO（投資戦略の最終承認者）
- 役割: 投資戦略の最終承認者
- 主要タスク: 理想リスクプロファイルの設定、ポートフォリオ変更の承認、取締役会報告

### 解決する課題

1. **情報の散在**: ファンド情報がExcel・メール・書類などに分散し、最新情報の把握が困難
2. **リスク把握の属人性**: ファンド評価が担当者の主観に依存し、標準化されていない
3. **ポートフォリオレベルの視点不足**: 個別ファンドは評価できても全体の相関・分散を定量的に分析できていない
4. **理想との乖離が不明**: 組織のリスク許容度に対し現状ポートフォリオがどう位置するか可視化できていない
5. **更新の遅延**: ファンド情報の更新が反映されず、古い情報で意思決定してしまうリスク

### 提供する価値

1. **一元管理**: 全ファンドの定量・定性・インフラ情報を1箇所で管理
2. **AI自動化**: DD・モニタリング・格付けをAIエージェントが自動実行
3. **標準化**: 評価軸を統一し、誰が実行しても同じ品質の分析が得られる
4. **リアルタイム**: スタイルドリフト・TE急変・資金流出を即座に検知
5. **最適化提案**: ポートフォリオ全体のリスク分散・相関最小化をAIが提案

### ソリューション

**AIエージェントモジュール方式**
- 各タスク（DD・サーベイランス・格付け・IC・最適化）ごとに専門AIエージェントを配置
- ファンドマネージャーは必要なモジュールを呼び出すだけで自動実行
- Claude API（tool_use）で構造化出力を直接取得し、DB保存

**技術構成**
- フレームワーク: Next.js 16 + TypeScript
- AI: Claude API（tool_use structured output）
- DB: Prisma v5 + SQLite
- スタイル: Tailwind CSS + Paper Design System

## ユーザーフロー

### 初回セットアップフロー

```
1. アプリ起動 → ダッシュボード
   ↓
2. ファンドダンプページへ移動
   ↓
3. CSVインポート or 手動入力でファンド情報を登録
   ↓
4. 各ファンドの定量・定性データを入力（タブで切替）
   ↓
5. ダッシュボードに戻り、登録済みファンドを確認
```

### 日常運用フロー（アナリスト）

```
1. ダッシュボードでアラート確認
   ↓
2a. 緊急アラートあり
    → サーベイランスページで詳細確認
    → 対象ファンドの状態を分析
    → アラートを「確認済」に変更
   ↓
2b. 新規ファンドのDD実施
    → DDエージェントページへ
    → ファンド情報をテキスト or ファイルで入力
    → AI分析実行（5軸評価）
    → DD履歴に保存
   ↓
3. ファンドダンプページで情報を更新
   ↓
4. 統合格付けページで評価を確認
```

### 月次レビューフロー（FM）

```
1. ダッシュボードでKPI確認（AUM・アラート件数・推奨ファンド数）
   ↓
2. 統合格付けページで全ファンドのスコア分布を確認
   ↓
3. 要観察・除外ファンドの詳細をファンドダンプで確認
   ↓
4. ポートフォリオ最適化ページへ移動
   ↓
5. 「AIに最適化を依頼」ボタンクリック
   ↓
6. AI提案を確認（リバランス案・リスク評価・相関分析）
   ↓
7. IC支援ページで議題を整理
   ↓
8. 投資委員会で提案
```

### DDフロー詳細

```
[DDエージェントページ]
1. テキスト入力 or ファイルアップロードを選択
   ↓
2. ファンド情報を入力（基本情報・戦略・トラックレコード等）
   ↓
3. 「DD分析を開始」ボタンクリック
   ↓
4. AI分析中（プログレスバー表示）
   - 運用哲学・プロセス
   - トラックレコード
   - チーム構成・安定性
   - 費用構造
   - ESG・サステナビリティ
   ↓
5. 結果表示
   - 総合評価（テキスト）
   - セクション別スコア（5軸）
   - 詳細評価（各セクションの評価コメント + フラグ）
   - 矛盾・不整合の検出
   - 主要リスク
   ↓
6. 履歴サイドバーに自動保存
   ↓
7. 「新規分析」で次のファンド or ダッシュボードに戻る
```

## 画面構成詳細

### 1. ダッシュボード（/）

**レイアウト**
```
[Header: タイトル + アラートバッジ]
[緊急アラートバナー]（条件付き表示）
[KPIメトリクスカード × 4]
  - 運用資産残高
  - 監視対象ファンド数
  - 緊急アラート件数
  - 推奨ファンド数
[AIエージェントモジュールカード × 5]
  - DDエージェント
  - サーベイランス
  - 統合格付け
  - IC支援
  - ポートフォリオ最適化
[ファンドユニバーステーブル]
  - ファンド名 | カテゴリ | AUM | スコア | 評価
```

**インタラクション**
- モジュールカードをクリック → 各機能ページへ遷移
- 緊急アラートバナーの「確認する」→ サーベイランスページへ
- ファンドテーブルの行クリック → ファンドダンプページへ（該当ファンド選択状態）

### 2. ファンドダンプ（/funds）

**レイアウト**
```
[Header]
[2カラムレイアウト]
  [左: ファンド一覧（3カラム分）]
    - ヘッダー: ファンド一覧 (N本) + [CSVエクスポート] [CSVインポート] [新規追加]
    - カード形式でファンド一覧表示
      - ファンド名・運用会社・評価バッジ・更新日

  [右: ファンド詳細（9カラム分）]
    - ヘッダー: ファンド名・運用会社・カテゴリ・評価 | 総合スコア
    - タブ: [定量リスクプロファイル] [インフラ・構造] [DDメモ]

    [定量リスクプロファイルタブ]
      - アップキャプチャ / ダウンキャプチャ（2カラム）
      - 定量指標カード × 8（4カラムグリッド）
        - アルファ・シャープ比・最大DD・ボラティリティ
        - ソルティノ比・ベータ・情報比率・TE
      - リターン分布カード
        - プラス月数・平均リターン・最良月
        - マイナス月数・平均リターン・最悪月
        - 勝率・カルマー比

    [インフラ・構造タブ]
      - 運用インフラカード
        - 取引所・PB・カストディアン・管理会社・監査法人
      - ファンド構造・チームカード
        - ドミサイル・法人形態・チームサイズ・キーPM
        - 流動性条件（ロックアップ・解約通知・ゲート）
      - 定性リスクチェックカード
        - 6項目のチェックリスト（✓ or ⚠）

    [DDメモタブ]
      - テキストエリア（モノスペースフォント）
```

**インタラクション**
- 左カラムのファンドカードクリック → 右カラムに詳細表示
- CSVインポートボタン → ダイアログ表示 → ファイル選択 → アップロード → DB保存 → 一覧更新
- CSVエクスポートボタン → CSV生成 → ダウンロード
- 新規追加ボタン → ダイアログ表示 → フォーム入力 → 保存 → 一覧更新

### 3. DDエージェント（/dd）

**レイアウト（入力画面）**
```
[Header]
[メインカード]
  [タブ: ファイルアップロード | テキスト直接入力]

  [ファイルアップロードタブ]
    - ドラッグ&ドロップエリア
    - アップロード済みドキュメント一覧（削除ボタン付き）

  [テキスト直接入力タブ]
    - テキストエリア（大）
    - 文字数カウンター

  [DD分析を開始ボタン]
```

**レイアウト（分析中画面）**
```
[カード中央配置]
  - ローディングアイコン（回転）
  - プログレスバー（0-100%）
  - 現在分析中のセクション名
  - セクション進捗チェックリスト
    ✓ 運用哲学・プロセス
    ● トラックレコード（分析中）
    ○ チーム構成・安定性
    ○ 費用構造
    ○ ESG・サステナビリティ
```

**レイアウト（結果画面）**
```
[Header: 「新規分析」ボタン]
[総合評価カード]（紫色背景）
  - AIアイコン + 総合評価テキスト

[セクション別スコアカード]
  - 5軸のスコアバー（0-100）
    - 運用哲学・プロセス: 85
    - トラックレコード: 72
    - チーム構成・安定性: 68
    - 費用構造: 55
    - ESG・サステナビリティ: 80

[詳細評価カード]
  - セクションごとのカード（5個）
    - セクション名 + アイコン
    - スコアバッジ（色分け: 緑/青/黄/赤）
    - 評価コメント（テキスト）
    - フラグバッジ（⚠警告）

[矛盾・不整合カード]（黄色背景、条件付き表示）
  - ⚠ 矛盾・不整合の検出 (N件)
  - リスト表示

[主要リスクカード]（赤色背景）
  - ⚠ 主要リスク
  - リスト表示

[右サイドバー: 分析履歴]（3カラム、折りたたみ可）
  - 履歴カード（クリックで再表示）
    - 日時
    - 総合評価の冒頭50文字
```

**インタラクション**
- テキスト入力 or ファイル選択 → 「DD分析を開始」有効化
- 分析開始 → API POST `/api/analyze/dd` → 分析中画面 → 結果画面
- 履歴カードクリック → 結果画面に再表示
- 「新規分析」→ 入力画面に戻る

## API エンドポイント

### POST /api/analyze/dd

**リクエスト**
```json
{
  "fundData": "ファンド名: XXX\n運用会社: YYY\n..."
}
```

**レスポンス**
```json
{
  "success": true,
  "data": {
    "overallConclusion": "総合評価テキスト",
    "sections": [
      {
        "title": "運用哲学・プロセス",
        "score": 85,
        "content": "評価コメント",
        "flags": ["警告1", "警告2"]
      }
    ],
    "keyRisks": ["リスク1", "リスク2"],
    "contradictions": ["矛盾1"]
  }
}
```

**リクエスト**
```json
{
  "currentPortfolio": [
    {"fundId": "f001", "weight": 0.2}
  ]
}
```

```json
{
  "success": true,
  "data": {
    "summary": "最適化サマリ",
    "recommendations": [
      {
        "fundId": "f001",
        "action": "increase",
        "currentWeight": 0.2,
        "proposedWeight": 0.25,
        "rationale": "理由"
      }
    ],
    "riskAssessment": {
      "diversificationScore": 75,
      "concentrationRisks": ["リスク1"],
      "correlationComment": "相関コメント"
    },
    "nextActions": ["アクション1"]
  }
}
```

### GET /api/funds

全ファンド情報を取得

CSVファイルをアップロードしてファンド情報を一括登録

### GET /api/dd-history

DD分析履歴を取得

## データモデル

### Fund（ファンド）

```typescript
{
  id: string
  name: string
  company: string
  category: "国内株式" | "外国株式" | ...
  style: "バリュー" | "グロース" | "ブレンド" | "インデックス"
  benchmark: string
  aum: number  // 億円
  nav: number  // 基準価額
  inceptionDate: string  // YYYY-MM-DD
  expenseRatio: number  // %
  isin: string
  rating: "推奨" | "継続" | "要観察" | "除外"
  ratingScore: number  // 0-100
  esgScore: number  // 0-100
}
```

### FundDump（ファンド詳細情報）

```typescript
{
  fundId: string
  lastUpdated: string
  updatedBy: string
  quantProfile: {
    upCapture: number  // %
    downCapture: number  // %
    posMonths: number
    negMonths: number
    avgPosReturn: number
    avgNegReturn: number
    bestMonth: number
    worstMonth: number
    calmarRatio: number
  }
  infrastructure: {
    exchanges: string[]
    primeBroker: string
    primeBrokerRating: string
    custodian: string
    adminCompany: string
    auditor: string
    isBig4Auditor: boolean
  }
  structure: {
    domicile: string
    legalEntity: string
    teamSize: number
    keyPm: string
```

### Claude API tool_use による構造化出力

DDエージェント・ポートフォリオ最適化では、Claude APIの `tool_use` 機能を使用して構造化JSONを直接取得。

```typescript
// src/lib/claude.ts
const tools = [{
  name: "generate_dd_report",
  description: "Generate structured DD report",
  input_schema: {
    type: "object",
    properties: {
      overallConclusion: { type: "string" },
      sections: {
        type: "array",
        items: {
          type: "object",
          properties: {
            title: { type: "string" },
            score: { type: "number" },
            content: { type: "string" },
            flags: { type: "array", items: { type: "string" } }
          }
        }
      },
      keyRisks: { type: "array", items: { type: "string" } },
      contradictions: { type: "array", items: { type: "string" } }
    }
  }

const response = await anthropic.messages.create({
  model: "claude-opus-4-6",
```

### Prisma + SQLite によるシンプルなデータ永続化

開発環境・デモ環境では SQLite を使用。本番環境では PostgreSQL への切替が容易。

### CSV インポート・エクスポート

ファンド情報をCSVで一括登録・エクスポート可能。既存Excelデータからの移行が容易。
