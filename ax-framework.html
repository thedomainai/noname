AX CONSULTING FRAMEWORK

AI Transformation 受注フレーム

クイックウィンで渇望を生み、段階的に深化させるランド&エクスパンド戦略。
各フェーズの出口が次フェーズへの自然な導線になる設計。

PHASE 0 — PREMISE

顧客の状態: 問題解決モードに入っている。AI readyになりたいが、何から始めるべきかわからない。AIがどれほどの変革をもたらすか実感がない。

再受注メカニズムの全体像

01
期待/渇望 1-2 weeks

顧客の実業務でクイックウィンを見せる。AIの可能性を肌で理解させ、「もっとやりたい」を引き出す。

DELIVER
  • 実データ・実業務でのAIデモ
  • 即日〜数日で動くプロトタイプ
EFFECT
  • 渇望の生成
  • 現場チャンピオンの獲得
RE-ORDER TRIGGER

「全社展開するには何が足りないか」の問いが自然発生する

02
実感 2-4 weeks

Phase 1で見せた価値をミニマムの自動化ツールとして実装・納品。デモを実務ツールに昇格させる。

DELIVER
  • ミニマムの自動化ツール開発
  • 現場が日常的に使える状態
EFFECT
  • 「業務に組み込まれた実績」の創出
  • 定量的な効果（時間削減等）
  • スイッチングコストの発生
RE-ORDER TRIGGER
  • 使用データから横展開候補が可視化
  • 「もっとこうしたい」の自然発生
  • 部門長への稟議材料が揃う
03
診断 2-4 weeks

業務・データ・組織の現状アセスメント。Phase 2の利用データを活用した実データ基盤の診断。

DELIVER
  • 組織AIレベル診断
  • ロードマップ策定
  • Phase 2利用データに基づく分析
EFFECT
  • 投資判断の材料
  • 経営層スポンサーの獲得
RE-ORDER TRIGGER

ロードマップ自体が後続フェーズの提案書として機能する

04
基盤 1-3 months

データ基盤 + セキュリティ/ガバナンス設計。AIが機能する土壌を組織に実装する。

DELIVER
  • データ基盤構築
  • セキュリティ/ガバナンス設計
  • 原理原則の集中教育
EFFECT
  • AIが機能する土壌の完成
  • 組織のAIリテラシー底上げ
RE-ORDER TRIGGER

基盤完成の瞬間に「何を載せるか」の議論が始まる

05
実装 3-6 months

適切なタスク選定からワークフロー設計、品質管理までを一気通貫で実行する。

DELIVER
  • タスク選定 + 優先順位付け
  • ワークフロー設計
  • 品質管理体制の構築
EFFECT
  • 実業務での定量的成果
  • タスク単位での品質向上
RE-ORDER TRIGGER

成果の定量化が他部門への横展開の根拠になる

06
自走 Retainer

改善サイクルが自律的に回る状態を構築。コンサル依存からの卒業と持続的な収益関係を両立。

DELIVER
  • 改善サイクルの仕組み化
  • 内製チームの育成
EFFECT
  • 顧客の自律的改善能力
  • コンサル卒業（依存解消）
REVENUE MODEL
  • 月次アドバイザリー
  • プラットフォーム利用料
CROSS-CUTTING
原理原則の教育
独立したステップではなく、全フェーズに通底するレイヤー。体験 → 判断材料 → 実践の中で段階的に浸透させる。
