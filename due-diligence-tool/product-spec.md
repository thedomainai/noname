# risk-profile-mvp プロダクトドキュメント

## プロダクトコンセプト

### ターゲットユーザー

**プライマリ**: FoFファンドマネージャー
- 役割: 投資候補ファンドのリスクプロファイル把握
- 主要タスク: 短時間でファンドのリスク特性を理解する
- ゴール: 定性情報からリスクを定量的に評価し、投資判断の材料にする

### 解決する課題

1. **定性情報の解釈コスト**: ファンド資料（PDF・テキスト）を読み込んでリスクを評価するのに時間がかかる
2. **評価の属人性**: アナリストによってリスク評価のポイントが異なり、標準化されていない
3. **リスクプロファイル作成の負担**: DD報告書にリスクプロファイルを書き起こす作業が煩雑
4. **即座の評価が困難**: 商談・電話中に即座にファンドリスクを評価できない

### 提供する価値

1. **即座のリスク評価**: テキスト入力 → 数秒で AI がリスクプロファイルを生成
2. **標準化された評価軸**: Claude Opus が一貫した基準でリスクを評価
3. **コピペ可能な出力**: 生成されたテキストをそのまま報告書に転記可能
4. **最小依存構成**: Next.js + Tailwind のみ、DB・認証なしでデプロイ可能

### ソリューション

**Single Page MVP**
- 1画面完結（入力 → 分析 → 結果）
- Claude Opus 4.6 API で高品質なリスクプロファイル生成
- インラインスタイルのみ（Tailwind CSS 直書き）
- 依存ライブラリ最小化（shadcn/ui 等のUI

ライブラリなし）

## ユーザーフロー

```
1. ページ表示
   - テキストエリアに入力欄が表示される
   ↓
2. ファンド情報を入力
   - 運用会社名・戦略・投資対象・リスク要因などを自由形式で入力
   - 500〜2000文字程度
   ↓
3. 「分析する」ボタンをクリック
   ↓
4. AI分析中（ローディング表示）
   - API POST /api/risk-profile
   - Claude Opus が分析実行（5〜15秒）
   ↓
5. 結果表示
   - リスクプロファイルテキストが表示される
   - セクション構成: リスク要約・主要リスク・リスク緩和策・総合評価
   ↓
6. 結果をコピー or 別のファンドを分析
   - 「新規分析」ボタンで入力画面に戻る
```

## 画面フロー

```
[ページ] (/)
  - 入力状態: テキストエリア + 分析ボタン
  - 分析中状態: ローディングインジケーター
  - 結果状態: リスクプロファイル + 新規分析ボタン
```

**リスクプロファイル構成例**

```
【リスク要約】
このファンドは新興国株式に集中投資する戦略であり、カントリーリスク・為替リスク・流動性リスクが主要なリスク要因です。

【主要リスク】
1. カントリーリスク: 投資先国の政治・経済不安定性により、株価が大きく変動する可能性
2. 為替リスク: 新興国通貨の対円レートの変動により、円換算リターンが減少する可能性
3. 流動性リスク: 市場の流動性低下時に、希望価格での売買が困難になる可能性
4. 集中リスク: 特定セクター・銘柄への集中により、ポートフォリオ全体が影響を受ける可能性

【リスク緩和策】
1. 分散投資: 複数国・複数セクターへの分散により、単一リスクの影響を軽減
2. ヘッジ戦略: 為替ヘッジを一部実施し、為替リスクを低減
3. 流動性管理: 流動性の高い銘柄を中心に組入れ、ゲート条項で急激な解約を制限

【総合評価】
リスク水準は中〜高レベル。投資先国の経済成長期待がリターンを牽引する一方、市場ショック時の下落幅が大きい点に留意が必要。
```

## 技術的特徴

### 最小構成

- フレームワーク: Next.js 15 + TypeScript
- スタイル: Tailwind CSS（インラインスタイルのみ、shadcn/ui なし）
- AI: Claude API（claude-opus-4-6、標準completion）
- DB: なし（ステートレス）
- 認証: なし

### API 実装

```typescript
// src/app/api/risk-profile/route.ts
import Anthropic from "@anthropic-ai/sdk";

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

export async function POST(request: Request) {
  const { fundData } = await request.json();

  const message = await anthropic.messages.create({
    model: "claude-opus-4-6",
    max_tokens: 2048,
    messages: [{
      role: "user",
      content: `以下のファンド情報からリスクプロファイルを生成してください。

【ファンド情報】
${fundData}

【出力形式】
以下のセクションで構成してください：
1. リスク要約（2〜3文）
2. 主要リスク（箇条書き3〜5項目）
3. リスク緩和策（箇条書き2〜4項目）
4. 総合評価（1〜2文）`
    }]
  });

  return Response.json({
    success: true,
    riskProfile: message.content[0].type === "text" ? message.content[0].text : ""
  });
}
```

## 今後の拡張予定

1. **履歴機能**: 過去の分析結果を保存・検索（要DB）
2. **比較機能**: 複数ファンドのリスクプロファイルを並べて比較
3. **定量指標連携**: Sharpe Ratio・最大DD 等の定量指標を自動取得して分析に反映
4. **PDF出力**: リスクプロファイルをPDF形式でエクスポート
5. **テンプレート**: ファンドカテゴリ別の入力テンプレート提供
6. **マルチ言語**: 英語・日本語の切替対応

## 制限事項

- 入力データの検証: 現状は簡易的なバリデーションのみ
- レート制限: Claude API のレート制限に依存（Tier 2: 40 req/min）
- 履歴なし: 分析結果は画面表示のみ、永続化しない
- 認証なし: 公開URLで誰でもアクセス可能（本番化時は認証必須）
