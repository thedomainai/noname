# DD Navigator - Product Requirements Document

## 1. Problem Statement（課題定義）

事業会社のM&A DDチームは、年間200件以上の候補企業スクリーニングと複数の並行DD案件管理を行っており、以下の非効率が発生している：

1. **ロングリスト管理の手作業化**: 候補企業情報の収集・更新・鮮度維持が属人化し、戦略フィット評価の基準が不明確
2. **Q&A重複チェックの工数**: DD中のQ&Aで重複質問が発生し、毎朝30分×2名の確認作業が必要。重複送信は対象企業への印象を損なうリスクがある
3. **開示資料の検索非効率**: VDR内の大量資料から必要情報を探すのに時間がかかり、参照元の追跡が困難
4. **事業計画整合性の手動検証**: プロジェクション（事業計画）の前提条件の整合性チェックが手作業で、ミスが見逃されやすい
5. **複数案件の並行管理の複雑性**: 各案件のフェーズ・タスク・期限・担当者の可視化が不十分

これらの非効率により、DD品質のバラつき、案件進行の遅延、意思決定に必要な情報の見落としが発生している。

## 2. Target Users（ターゲットユーザー）

### プライマリーユーザー

M&A担当チームメンバー: 事業会社コーポレート部門の4-6名チーム

- チームリーダー（VP級）: 案件全体の統括、最終判断、外部アドバイザー管理
- シニアメンバー（2-3名）: DD設計・実行、Q&A統括、レポート作成
- ジュニアメンバー（1-2名）: ロングリスト作成、資料整理、Q&A管理

### ユーザー特性

- DD経験が浅いメンバーも含まれる（「どこを見ればいいかわからない」）
- 並行して複数案件を担当（年間45件のIM検討、うち数件がDD実行）
- 外部アドバイザー（BDD/FDD/LDD専門家）との協業が必要

## 3. Value Proposition（提供価値）

DD Navigatorは、M&A DDプロセスの構造化と自動化により、以下の価値を提供する：

1. **DD品質の標準化**: 経験の浅いメンバーでも一定品質のDDを実行できるガイド機能
2. **工数削減**: ロングリスト管理・Q&A重複チェック・資料検索の自動化により、年間500時間以上の工数削減
3. **意思決定の高速化**: 案件情報・DD結果の一元管理により、経営判断に必要な情報へのアクセスを即座に実現
4. **リスク低減**: Q&A重複防止・事業計画整合性チェックにより、対象企業への印象悪化と判断ミスを防止
5. **ナレッジ蓄積**: 過去案件のDD結果・Q&Aパターンをデータベース化し、チーム全体の学習曲線を短縮

## 4. Functional Requirements（機能要件）

### Must Have（v1必須）

#### F1: ロングリスト管理

- **F1.1 候補企業の登録・編集**: 企業名、業種、地域、売上規模、従業員数、戦略フィット評価（5段階）を登録
- **F1.2 フィルタリング・ソート**: 業種・地域・売上規模・戦略フィット評価で絞り込み、更新日順・評価順でソート
- **F1.3 鮮度管理**: 最終更新日を表示し、90日以上更新されていない企業にアラート表示
- **F1.4 CSV一括インポート**: 既存のExcelロングリストからCSVインポートで一括登録

#### F2: DDプロジェクト管理

- **F2.1 プロジェクト作成**: 案件名、対象企業、想定クロージング日、担当チームメンバー（リーダー・メンバー）を登録
- **F2.2 フェーズ管理**: 9フェーズ（ノンネーム→NDA→IM分析→LOI→MOU→DD実行→DD統合→SPA→クロージング）のステータス管理
- **F2.3 DD領域管理**: BDD/FDD/LDD/TDD/HRDD/ITDDの6領域について、担当者（内部・外部アドバイザー）と進捗状況を記録
- **F2.4 タスク管理**: フェーズごとのタスクリスト（チェックリスト形式）を表示し、完了・未完了を管理
- **F2.5 ダッシュボード**: 全案件の一覧表示（フェーズ別・担当者別フィルタ可能）、期限が近い案件のアラート表示

#### F3: Q&A管理

- **F3.1 Q&A登録**: プロジェクト内でQ&Aを作成（質問文、回答文、DD領域タグ、ステータス［質問準備中・送信済み・回答受領済み］）
- **F3.2 重複チェック**: 質問文を入力中、過去の類似質問を自動サジェスト（キーワード一致ベース）
- **F3.3 Q&Aリスト**: プロジェクト内の全Q&Aを一覧表示（領域別・ステータス別フィルタ可能）
- **F3.4 エクスポート**: Q&Aリストをテキストまたはスプレッドシート形式でエクスポート

#### F4: 開示資料管理

- **F4.1 資料アップロード**: プロジェクト内でPDF・Excel・PowerPointをアップロード（ファイル名・カテゴリ・アップロード日を記録）
- **F4.2 カテゴリ分類**: 資料を「財務諸表」「事業計画」「契約書」「人事資料」等のカテゴリで分類
- **F4.3 ファイル検索**: ファイル名・カテゴリで検索可能
- **F4.4 資料リスト**: プロジェクト内の全資料を一覧表示（カテゴリ別・日付別ソート可能）

#### F5: 認証・権限管理

- **F5.1 ユーザー登録・ログイン**: メールアドレス + パスワード認証（Magic Link方式）
- **F5.2 チーム管理**: チームメンバーの招待（メール送信）、メンバー一覧表示、ロール設定（管理者・メンバー）
- **F5.3 プロジェクトアクセス制御**: プロジェクトごとにアクセス可能なメンバーを設定（デフォルトはチーム全員）

### Should Have（v1で実装すべき）

#### F6: 事業計画整合性チェック（ルールベース）

- **F6.1 前提条件の抽出**: アップロードされた事業計画（Excel）から売上成長率・EBITDA率・設備投資額等の数値を抽出
- **F6.2 整合性ルールの定義**: 「売上成長率 > 市場成長率 + 5%」等のルールを事前定義
- **F6.3 チェック実行**: ルールに違反する項目をハイライト表示し、アラート通知

#### F7: DDチェックリストテンプレート

- **F7.1 テンプレート登録**: DD領域（BDD/FDD/LDD等）ごとにチェックリスト項目を登録（「過去3年の財務諸表を確認」等）
- **F7.2 プロジェクトへの適用**: 新規プロジェクト作成時にテンプレートを選択し、タスクリストに展開
- **F7.3 カスタマイズ**: プロジェクトごとにチェックリスト項目を追加・削除可能

#### F8: レポート生成

- **F8.1 DD進捗レポート**: プロジェクトごとのDD進捗状況（各領域の完了率・未完了タスク・期限超過タスク）をPDF出力
- **F8.2 Q&Aサマリ**: プロジェクト内の全Q&Aを領域別にまとめたPDFを生成

### Could Have（余裕があれば実装）

#### F9: 資料内容検索（全文検索）

- **F9.1 PDF/Excelのテキスト抽出**: アップロードされた資料からテキストを抽出しインデックス化
- **F9.2 キーワード検索**: 資料内のキーワードで検索し、該当ページ・セルを表示

#### F10: 過去案件ナレッジ検索

- **F10.1 類似案件の検索**: 業種・規模が類似する過去案件を検索
- **F10.2 過去Q&Aの参照**: 類似案件で発生したQ&Aを表示

#### F11: ダッシュボードのカスタマイズ

- **F11.1 ウィジェット配置**: ユーザーが表示する情報（期限が近い案件・未読Q&A等）を選択・並び替え可能

### Won't Have（今回は実装しない）

#### F12: LLM/AI機能

- **理由**: v1ではルールベースの自動化に注力。LLMを使った「NotebookLM的な資料問い合わせ」「事業計画の自動分析」は将来の拡張として扱う
- **将来の拡張候補**: 資料の要約生成、Q&A自動ドラフト、事業計画の異常値検出

#### F13: VDR統合

- **理由**: Intralinks等のVDRとのAPI連携は初期スコープ外。手動アップロードで対応
- **将来の拡張候補**: VDRからの自動同期、VDR内の資料更新通知

#### F14: 外部アドバイザーとのコラボレーション

- **理由**: 外部アドバイザー（BDD/FDD専門家）への招待・権限管理は今回スコープ外
- **将来の拡張候補**: ゲストユーザー機能、外部アドバイザー向けの限定ビュー

#### F15: モバイルアプリ

- **理由**: v1はWebアプリケーションのみ。モバイル対応は将来検討
- **将来の拡張候補**: レスポンシブデザイン対応、PWA化

## 5. User Stories（ユーザーストーリー）

### ロングリスト管理

**US-1**: As a **ジュニアメンバー**, I want to **候補企業を一括インポートできる**, so that **既存Excelリストから効率的に移行できる**

**US-2**: As a **シニアメンバー**, I want to **戦略フィット評価でフィルタリングできる**, so that **優先度の高い候補企業に集中できる**

**US-3**: As a **チームリーダー**, I want to **90日以上更新されていない企業にアラートが表示される**, so that **ロングリストの鮮度を維持できる**

### DDプロジェクト管理

**US-4**: As a **シニアメンバー**, I want to **新規プロジェクトを作成し、フェーズとDD領域を設定できる**, so that **案件の構造を明確に定義できる**

**US-5**: As a **チームリーダー**, I want to **全案件のダッシュボードで進捗を一覧できる**, so that **複数案件の優先順位を判断できる**

**US-6**: As a **ジュニアメンバー**, I want to **フェーズごとのタスクリストが表示される**, so that **何をすべきかが明確にわかる**

**US-7**: As a **シニアメンバー**, I want to **期限が近い案件にアラートが表示される**, so that **遅延リスクを早期に認識できる**

### Q&A管理

**US-8**: As a **ジュニアメンバー**, I want to **質問文を入力中に類似質問がサジェストされる**, so that **重複質問を防止できる**

**US-9**: As a **シニアメンバー**, I want to **Q&Aのステータスを管理できる**, so that **どの質問が未回答かを追跡できる**

**US-10**: As a **チームリーダー**, I want to **Q&Aリストを領域別にエクスポートできる**, so that **外部アドバイザーへの共有が容易になる**

### 開示資料管理

**US-11**: As a **ジュニアメンバー**, I want to **資料をカテゴリ別にアップロードできる**, so that **後で検索しやすくなる**

**US-12**: As a **シニアメンバー**, I want to **資料をファイル名・カテゴリで検索できる**, so that **必要な情報に素早くアクセスできる**

**US-13**: As a **チームリーダー**, I want to **プロジェクト内の全資料を一覧表示できる**, so that **情報の網羅性を確認できる**

### 認証・権限管理

**US-14**: As a **チームリーダー**, I want to **チームメンバーを招待できる**, so that **メンバー全員が同じ情報にアクセスできる**

**US-15**: As a **チームリーダー**, I want to **プロジェクトごとにアクセス可能なメンバーを制限できる**, so that **機密情報を保護できる**

### 事業計画整合性チェック

**US-16**: As a **シニアメンバー**, I want to **アップロードした事業計画の整合性を自動チェックできる**, so that **前提条件の矛盾を見逃さない**

**US-17**: As a **チームリーダー**, I want to **整合性違反の項目がハイライト表示される**, so that **重点的にレビューすべき箇所が明確になる**

### DDチェックリストテンプレート

**US-18**: As a **シニアメンバー**, I want to **DD領域ごとのチェックリストテンプレートを登録できる**, so that **標準的なDD手順を再利用できる**

**US-19**: As a **ジュニアメンバー**, I want to **新規プロジェクトでテンプレートを適用できる**, so that **経験が浅くても必要な確認項目を漏らさない**

### レポート生成

**US-20**: As a **チームリーダー**, I want to **DD進捗レポートをPDF出力できる**, so that **経営層への報告資料を効率的に作成できる**

**US-21**: As a **シニアメンバー**, I want to **Q&AサマリをPDF出力できる**, so that **外部アドバイザーへの情報共有が容易になる**

## 6. Success Criteria（成功基準）

### 定量的KPI

#### 利用指標

- **K1**: プロジェクト登録数：月間10件以上（SmartHR想定: 年間45件のIM検討）
- **K2**: Q&A登録数：プロジェクトあたり50件以上
- **K3**: 資料アップロード数：プロジェクトあたり200ファイル以上
- **K4**: 週次アクティブユーザー数：チームメンバーの80%以上

#### 効率化指標

- **K5**: Q&A重複チェック工数：30分/日 → 5分/日（83%削減）
- **K6**: ロングリスト更新頻度：90日 → 30日（鮮度向上）
- **K7**: DD進捗レポート作成時間：2時間 → 15分（88%削減）

#### 品質指標

- **K8**: Q&A重複送信の発生件数：0件（重複チェック機能により完全防止）
- **K9**: DD必須項目の漏れ：0件（チェックリスト機能により防止）

#### ユーザー満足度

- **K10**: NPS（Net Promoter Score）：40以上（初回ユーザーサーベイ）
- **K11**: ユーザー継続率：90%以上（3ヶ月後）

### 定性的成功基準

- **S1**: ジュニアメンバーが「DDで何を確認すべきかが明確になった」と評価する
- **S2**: シニアメンバーが「複数案件の並行管理が容易になった」と評価する
- **S3**: チームリーダーが「経営層への報告準備が迅速化した」と評価する

## 7. Scope Boundaries（スコープの境界）

### やること（In Scope）

1. **DD業務の構造化**: ロングリスト管理、プロジェクト管理、Q&A管理、資料管理の基本機能
2. **ルールベースの自動化**: Q&A重複チェック、事業計画整合性チェック（定義済みルール）、鮮度アラート
3. **標準化支援**: DDチェックリストテンプレート、レポート生成
4. **チーム内コラボレーション**: 認証、チーム管理、プロジェクトアクセス制御

### やらないこと（Out of Scope）

1. **LLM/AI高度機能**: 資料の要約生成、Q&A自動ドラフト、事業計画の自動分析（将来拡張）
2. **VDR統合**: Intralinks等の外部VDRとのAPI連携（将来拡張）
3. **外部アドバイザー管理**: 外部専門家への招待・権限管理（将来拡張）
4. **モバイルアプリ**: スマートフォン専用アプリ（v1はWebアプリのみ）
5. **複雑なワークフロー自動化**: 承認フロー、通知の細かいカスタマイズ（v1はシンプルなアラートのみ）
6. **財務モデリング機能**: バリュエーション計算、シナリオ分析（DDツールとしてのスコープ外）

### 技術的境界

- **認証方式**: Magic Link（メールアドレス + パスワードレス）のみ。SSOは今回スコープ外
- **ファイルストレージ**: 5GB/チーム（初期上限）。大容量ファイル（動画等）は対象外
- **ブラウザサポート**: Chrome、Safari、Edge の最新版のみ。IE11は非対応
- **セキュリティ**: チーム単位のアクセス制御。SOC2/ISO27001準拠は今回スコープ外

## 8. Non-Functional Requirements（非機能要件）

### パフォーマンス

- **NFR-1**: ページ読み込み時間：3秒以内（初回ロード）、1秒以内（ページ遷移）
- **NFR-2**: Q&A重複チェック：入力中のリアルタイムサジェスト（200ms以内）
- **NFR-3**: ファイルアップロード：10MB以下のファイルで5秒以内に完了

### スケーラビリティ

- **NFR-4**: 同時接続ユーザー数：50名まで対応（初期想定）
- **NFR-5**: プロジェクト数：チームあたり100件まで
- **NFR-6**: Q&A数：プロジェクトあたり1,000件まで

### セキュリティ

- **NFR-7**: 認証：Magic Link方式（パスワードレス）
- **NFR-8**: 通信暗号化：HTTPS必須
- **NFR-9**: データ暗号化：保存時の暗号化（at rest）
- **NFR-10**: アクセス制御：プロジェクトごとのメンバー制限

### 可用性

- **NFR-11**: サービス稼働率：99.5%以上（月間ダウンタイム3.6時間以内）
- **NFR-12**: バックアップ：日次自動バックアップ、30日間保持

### ユーザビリティ

- **NFR-13**: モバイル対応：レスポンシブデザイン（タブレット・スマートフォン表示対応）
- **NFR-14**: アクセシビリティ：WCAG 2.1 Level AA準拠
- **NFR-15**: 多言語対応：日本語のみ（v1）。英語対応は将来拡張

## 9. Constraints（制約）

### 技術的制約

- **C1**: Webアプリケーションとして実装（ブラウザベース）
- **C2**: Next.js App Router を使用（技術スタック統一のため）
- **C3**: Supabase を認証・データベースとして使用
- **C4**: ファイルストレージは Supabase Storage を使用

### ビジネス制約

- **C5**: 開発期間：3ヶ月以内にv1リリース
- **C6**: 初期ユーザー：SmartHR M&Aチーム（6名）での検証
- **C7**: 課金モデル：v1は無料提供（検証フェーズ）。v2以降で有料化を検討

### 法的制約

- **C8**: 個人情報保護：対象企業の機密情報を扱うため、適切なアクセス制御が必須
- **C9**: データ保管場所：日本国内（Supabase Tokyoリージョン）

## 10. Dependencies（依存関係）

### 外部依存

- **D1**: Supabase（認証・データベース・ストレージ）の安定稼働
- **D2**: Vercel（ホスティング）の安定稼働

### 内部依存

- **D3**: SmartHR M&Aチームからのフィードバック（v1機能検証のため）
- **D4**: DD領域チェックリストのドメイン知識提供（BDD/FDD/LDD専門家）

### 技術的依存

- **D5**: PDF/Excelのテキスト抽出ライブラリ（全文検索機能実装時）
- **D6**: LLM API（将来の拡張機能実装時）

## 11. Risks & Mitigation（リスクと対策）

### リスク1: DD経験の浅いメンバーがチェックリストを適切に使いこなせない

- **影響度**: 高（DD品質低下）
- **対策**: オンボーディングドキュメント作成、初回利用時のガイドツアー機能

### リスク2: Q&A重複チェックの精度が低く、誤検知または見逃しが発生する

- **影響度**: 中（工数削減効果の低下）
- **対策**: キーワード一致アルゴリズムの改善、ユーザーフィードバックによる学習

### リスク3: 大量ファイルアップロード時のパフォーマンス低下

- **影響度**: 中（ユーザー体験の悪化）
- **対策**: ファイルサイズ上限設定（10MB/ファイル）、バッチアップロード機能の提供

### リスク4: 外部アドバイザーとの情報共有が煩雑で、ツール外でのやり取りが発生する

- **影響度**: 中（ツール利用率の低下）
- **対策**: v1ではエクスポート機能で対応。v2で外部ユーザー招待機能を検討

### リスク5: LLM/AI機能への期待が高く、v1のルールベース機能では満足度が低い

- **影響度**: 低（v1は検証フェーズのため）
- **対策**: ロードマップを明示し、v2でのAI機能拡張を伝える
