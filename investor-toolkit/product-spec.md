# retail-investor プロダクトドキュメント（日本株個人投資家向け12ツール）

## プロジェクト概要

### 全体コンセプト

**日本株個人投資家の「痒いところ」に特化した実用ツール群**

既存の証券会社ツールやスクリーニングサービスではカバーしきれない、日本株特有の細かい分析・集計ニーズに応える12カテゴリのWebアプリケーション集。

### ターゲットユーザー

- **プライマリ**: 日本株で能動的に銘柄選択・ポートフォリオ管理を行う個人投資家（年間売買回数 10回以上、保有銘柄 5銘柄以上）
- **セカンダリ**: 配当投資・株主優待投資などテーマ型投資を行う個人投資家

**ペルソナ**:
- 年齢: 30〜60代
- 投資経験: 3年以上
- 投資スタイル: バリュー投資・配当投資・株主優待クロス・成長株投資など多様
- 痛み: 証券会社ツールの機能不足、Excel集計の手間、税務計算の煩雑さ、情報収集の分散

### 解決する課題

| 課題カテゴリ | 痛みの内容 | 既存ツールの限界 |
|---|---|---|
| **税務最適化** | 損益通算・取得単価管理・NISA枠管理が煩雑 | 証券会社の取引履歴は単なる一覧。損益通算プラン提案なし |
| **株主優待管理** | 優待クロス取引の逆日歩リスク・在庫・コスト計算が複雑 | 優待検索サイトはあるが、クロス取引コスト試算ツールなし |
| **決算分析** | 進捗率・修正傾向の定量把握が困難 | アナリストレポートは有料。個人投資家向け分析ツールなし |
| **ポートフォリオリスク** | セクター集中・相関・為替感応度の可視化が手動 | 証券会社ツールは時価評価と損益のみ。リスク分析機能なし |
| **情報収集** | EDINET・TDnet・空売り残高・大量保有報告書が分散 | 各サイトを個別巡回。統合ビューアなし |

### 提供する価値

1. **時間節約**: Excel手動集計 → ワンクリック可視化（週次作業 2時間 → 10分）
2. **判断精度向上**: 勘と経験 → 定量データに基づく意思決定
3. **税務リスク低減**: 確定申告ミス防止、損益通算最適化で税負担軽減
4. **学習効果**: 過去の自分の売買振り返り、感情と成績の相関分析

### ソリューション

**12カテゴリ × 複数機能 = 50+ 実用ツール**

各カテゴリは独立したNext.jsアプリケーションとして実装。共通のデザインシステム（Paper Design System）を使用し、一貫したUXを提供。

## 1. 株主優待特化ツール（01-shareholder-benefits）

### コンセプト

**ターゲット**: 株主優待クロス取引を行う個人投資家、優待生活を目指す長期投資家

**解決する課題**:
- 優待クロス取引の逆日歩リスク・在庫・コストを統合管理
- 優待の実質利回り（換金率込み）計算が複雑
- 継続保有条件（1年以上）のカウントダウン管理が手動
- 優待廃止・改悪の情報収集が遅れる

**提供する価値**:
- 権利落ち日カレンダーで取引タイミングを可視化
- 優待品の市場価値を組み込んだ実質利回り計算
- 継続保有条件の自動追跡
- 優待廃止・改悪の自動アラート（TDnet監視）
- 「優待だけで生活費をいくら賄えるか」のシミュレーション

**ソリューション**: 5つのタブで構成されたダッシュボード

### ユーザーフロー

```
1. ダッシュボードトップ表示
   ↓
2. タブ選択（5つの機能から選択）
   - 優待カレンダー: 権利落ち日・在庫・逆日歩リスク確認
   - 利回り計算: 優待品の換金率込みで実質利回り算出
   - 保有条件: 継続保有条件のカウントダウン表示
   - 廃止アラート: 適時開示から優待変更を検知
   - シミュレーター: 何銘柄でいくら投資すれば生活費を賄えるか試算
   ↓
3. 各機能を使用
   ↓
4. 結果を保存 or エクスポート（CSV/PDF）
```

### 画面フロー

```
[ダッシュボードトップ]
  ├─ [優待カレンダー] ← デフォルト表示
  ├─ [利回り計算]
  ├─ [保有条件]
  ├─ [廃止アラート]
  └─ [シミュレーター]
```

### 画面構成詳細

```
[タブナビゲーション（5タブ）]
  📅 優待カレンダー | 🧮 利回り計算 | ⏱️ 保有条件 | ⚠️ 廃止アラート | 💰 シミュレーター

[メインコンテンツエリア]

[フッター]
```

#### 優待カレンダー（CrossTradingCalendar）

**機能**: 権利落ち日を月次カレンダー形式で表示。各銘柄の在庫・逆日歩リスクを色分け。

**レイアウト**:
```

  - 各日付セルに権利落ち銘柄を表示
  - 色分け: 在庫潤沢（緑）/ 在庫不足（黄）/ 高逆日歩リスク（赤）

[銘柄詳細テーブル]

[フィルター]
  - 月選択、逆日歩リスクレベル、優待カテゴリ（食品・商品券・自社製品等）
```

**データ例（モック）**:
```typescript

  inventory: "潤沢",
  gynoDaikoRisk: "低",

```

#### 利回り計算（YieldCalculator）

**機能**: 優待品の市場価値（換金率）を組み込んだ実質利回り計算。

**レイアウト**:
```
[入力フォーム]
  - 銘柄コード（自動補完）
  - 購入単価
  - 優待品の換金率（%）: デフォルト 80%（金券ショップ相場）

[計算結果]
  - 配当利回り: X.X%
  - 優待利回り（換金率込み）: Y.Y%
  - 実質総合利回り: Z.Z%
  - 年間受取額（円）

[比較表]
  - 換金率 50% / 80% / 100% での利回り比較
```

**計算式**:
```
配当利回り = 年間配当 / 購入単価
優待利回り = (優待品の額面 × 換金率) / 購入単価
実質総合利回り = 配当利回り + 優待利回り
```

```
[アラート設定]
```

```typescript
  status: "あと70日",

```

#### 廃止アラート（CancellationAlerts）

**機能**: TDnet適時開示から優待廃止・改悪を自動検知してアラート表示。

**レイアウト**:
```
[アラート一覧]
  - 最新順に表示

    - 銘柄名・コード
    - 変更内容（廃止 or 改悪詳細）
    - 発表日
```

#### シミュレーター（LifestyleSimulator）

**機能**: 何銘柄にいくら投資すれば、生活費をどれだけ優待で賄えるかの試算。

**レイアウト**:
```

[銘柄選択（複数）]

[シミュレーション結果]

  - 月間カバー額

  - 残り不足額

[グラフ]
  - 月別優待入金予定（積み上げ棒グラフ）
```

- 配当性向・DOEの長期トレンドが見えない
- 増配連続年数の日本版「配当貴族」探索が困難
- 特別配当を含めた「実力配当」の把握が手動

**提供する価値**:
- 月次配当カレンダーで入金月平準化をサポート
- 配当性向・DOEの10年トレンド可視化
- 15年・20年連続増配銘柄のスクリーニング
- 配当再投資シミュレーション（DRIP効果検証）
- 累進配当宣言企業のトラッキング

## 3. 税務・確定申告支援（03-tax-filing）

### コンセプト

**ターゲット**: 特定口座・一般口座・NISAを併用する個人投資家

**解決する課題**:
- 損益通算の最適順序が分からない（年末駆け込み売却の判断）
- 複数回購入した銘柄の取得単価管理が煩雑（総平均法/移動平均法）
- NISA枠の残枠・期限管理がExcel手動
- 外国税額控除の計算が複雑
- 株式分割・合併後の簿価補正がミスしやすい

**提供する価値**:
- 損益通算プランナー: 年末に向けた売却順序提案
- 取得単価計算機: 総平均法・移動平均法で自動計算
- NISA枠消化トラッカー: 成長投資枠・つみたて投資枠の残枠管理
- 確定申告要否判定: 口座種別と取引状況から申告要否を自動判定
- 株式分割後の単価補正: 分割比率入力で自動再計算

### ユーザーフロー

```
1. ツール選択画面表示（5つのサブツール）
   ↓
2. サブツール選択
   - 損益通算プランナー
   - 取得単価計算機
   - NISA枠消化トラッカー
   - 確定申告要否判定
   - 株式分割後の単価補正
   ↓
3. 各ツールで計算・判定
   ↓
4. 結果エクスポート（PDF/CSV）
```

### 画面構成（page.tsx）

**レイアウト**:

## 4. 決算・IR分析（04-earnings-ir）

### コンセプト

**ターゲット**: 決算発表を重視する個人投資家、四季報投資家

**解決する課題**:
- 会社予想 vs アナリスト予想 vs 実績のかい離（サプライズ）が定量化されていない
- 第1Q〜第3Qの進捗率を過去と比較する手段がない
- 上方/下方修正の頻度・幅を定量評価できない
- 決算短信のテキスト変化（前期と今期の比較）が手動
- IR説明会資料のアップロードを見逃す

**提供する価値**:
- 決算サプライズ検知器: 予想vs実績のかい離を数値化
- 進捗率トラッカー: 第1Q〜第3Qの予算進捗率を過去実績と比較
- 上方/下方修正 頻度・幅の分析: 「保守的見通しを出しやすい会社」を定量評価
- 決算短信テキスト差分ビューア: 前期と今期の定性コメント変化を色分け表示
- IR説明会資料更新通知
- セグメント損益トレンド可視化

### ユーザーフロー

```
1. ダッシュボード表示
   ↓
2. 機能選択（6つ）
   - 決算サプライズ検知器
   - 進捗率トラッカー
   - 上方/下方修正分析
   - テキスト差分ビューア
   - IR資料更新通知
   - セグメント損益トレンド
   ↓
3. 銘柄選択 or ウォッチリスト一括表示
   ↓
4. 分析結果確認
   ↓
5. アラート設定 or エクスポート
```

### 画面構成

**ダッシュボード**:
```
[ヘッダー]
  - タイトル: 「決算・IR分析」
  - 今週の決算発表予定銘柄（保有銘柄のみ）

[機能カード（6つ）]
  - 各機能へのリンク + 最新アラート件数バッジ
```

## 5. 大株主・需給分析（05-major-shareholders）

### コンセプト

**ターゲット**: 需給・需要背景を重視する投資家、ファンダメンタル+需給併用投資家

**解決する課題**:
- 大量保有報告書（5%ルール）の銘柄別追跡が手動
- 自社株買いの進捗率・取得単価が分散情報
- 大株主欄の変化（機関投資家のポジション変化）を定点観測できない
- 信用買い残・売り残の急変を見逃す

### Phase 1（MVP）開発優先順位

1. **03. 税務最適化ツール**: 損益通算プランナー + NISA枠トラッカー（2機能のみ）
2. **01. 株主優待ツール**: 優待カレンダー + 利回り計算機（2機能のみ）
3. **04. 決算進捗率トラッカー**: 進捗率トレンド分析のみ

Phase 1では「最も痛い部分」の3カテゴリ・7機能に絞り込んでMVPリリース。ユーザーフィードバックを得てから残り9カテゴリを順次追加。

### マネタイゼーション戦略

- **Freemium モデル**: 基本機能は無料、高度機能（AI分析・アラート通知・エクスポート）は有料
- **月額課金**: ¥980/月 or ¥9,800/年（年払いで2ヶ月無料）
- **法人プラン**: ¥49,800/年（社内複数ユーザー・API提供）

## まとめ

**retail-investor ツール群**は、日本株個人投資家の「既存ツールではカバーされない細かいニーズ」に特化した実用Webアプリケーション集。

- **12カテゴリ・50+ 機能**: 税務・優待・決算・需給・ポートフォリオリスク等を網羅
- **Paper Design System**: 一貫したデザイン言語
- **モックデータ駆動**: Phase 1では実データ連携なしでUI/UX検証
- **独立デプロイ**: 各カテゴリは独立したNext.jsアプリ（ポート分離）

Phase 1では優先度 ★★★ の3カテゴリに絞り込み、ユーザーフィードバックを得ながら段階的に機能拡張する戦略。
