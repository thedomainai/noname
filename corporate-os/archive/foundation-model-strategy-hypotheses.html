Section 6

三極構造の仮説

2029年の市場は Single Winner ではなく、三極分化する。利益プール・配布面・調達要件・プロトコルの違いが Winner-take-all を阻む。

OpenAI
Premium Intelligence Platform
Standalone AI experience / Frontier developer mindshare / Consumer × Enterprise 統合 / Government channel / Outcome-based monetization
Anthropic
Trusted Serious-Work Layer
High-stakes enterprise / Regulated industries / Cloud-neutral / Standards-first / "Safest serious choice"
Google
Bundled Default Fabric
Seat expansion / Frictionless adoption / Daily use 埋め込み / Cloud-native orchestration / Multi-model on Google infra
WHY NOT WINNER-TAKE-ALL

1. 利益プールが違う

Consumer, Enterprise, Public Sector, Developer, Commerce で勝ち方が異なる

2. 配布面が違う

Standalone AI App vs Existing Surface vs Embed/Partner

3. 調達要件が違う

Best Model vs Trusted Model vs Bundled Default

4. Semi-open プロトコル

MCP / A2A / ADK の世界では完全封鎖型 lock-in で勝ち切れない

5. 能力の Commoditize

差は Distribution, Context, Control, Workflow に移る

Section 9

Do / Don't

どのシナリオでも有効な Robust Bets と、どのシナリオでも危険な No-Go を整理する。

ROBUST BETS — DO
  • AGI-native Operating Model Design を上流商品に持つ
  • 提案だけで終わらず 1つの実装楔 を必ず持つ
  • 楔は Permissioned Context / Policy Routing / Evals / Industry Workflow のいずれかに寄せる
  • MCP / A2A 時代を前提に Managed Interoperability Layer を作る
  • 業界ごとに Task → Role → Team → Function の再設計テンプレートを蓄積
  • 最終的に Transition OS と Industry Operating Layer に上がる
NO-GO — DON'T
  • 汎用 AI 戦略コンサル — OpenAI/Google/大手コンサルが混む
  • 汎用社内チャット / Copilot — ChatGPT/Gemini/Claude に飲まれる
  • Thin Router — クラウド同梱・内製で圧縮
  • Thin RAG / Connector — Company Knowledge/MCP で吸収
  • 汎用 No-code Agent Builder — Frontier/Gemini Enterprise と正面衝突
  • 楔が本丸に資産化しない受託 — Future OS の知見が蓄積しない

Section 11

この仮説が外れうる5つの理由

仮説は仮説である。以下の変数が動けば、フレーム全体の再構築が必要になる。

1

モデル能力差が想定以上に縮小

利益が上位レイヤーにもっと早く移る

2

逆に能力差が拡大し続ける

Premium frontier provider がより強くなる

3

規制が急に厳しくなる / 緩い

3社すべての優位構造が変わりうる

4

Distribution owner が別に現れる

OS / Browser / Enterprise suite 側がさらに強化

5

大企業の導入速度が想定より遅い

Enterprise runtime より co-pilot 的用途が長引く

この枠組みが有用なのは、「どの会社が勝つか」だけでなく「どのレイヤーがどの会社に取られやすいか」を整理できるから。そして重要なのは、勝者予想よりも、勝者が誰でも残る white space を見つけること。
