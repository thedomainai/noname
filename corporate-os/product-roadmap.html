Corporate OS — Product Strategy

System of Governance

AI時代の企業基幹システム プロダクトロードマップ
April 2026
THESIS

ERPが「取引の文法」を標準化してコンピュータ時代の基幹になったように、AI時代には「判断の文法」を標準化する基幹システムが必要になる。

パチョーリが532年前に複式簿記を体系化したように、我々は13次元×5要素で組織の判断を体系化する。

センターピン — 全Phaseを貫く不変の核

Decision Ledger

すべての重要判断に authority, accountability, traceability を付与し、判断の整合性が検証可能であること。データの蓄積源であり、ネットワーク効果の基盤であり、13次元の実装起点。

ERP : 取引の記録 = 本システム : 判断の統治
Phase 0 — 即日〜3ヶ月
Market Entry

制度設計コンサルティング

13次元を使った組織診断・設計をサービスとして提供する。ソフトウェアはまだ作らない。

提供価値

AI時代の組織設計を13次元で診断し、destination（将来像）とtransition（移行設計）を提供。顧客企業の生の痛みを理解し、プロダクト設計の具体的なユースケースを獲得する。

購買者

CEO / COO直下のtransformation office、経営企画部長

→

最初のキャッシュフロー。13次元のどこに需要が集中するかの実データ。プロダクト仕様の素材。

Phase 1 — 3〜9ヶ月
Build MVP

権限配分エンジン

AIエージェントと人間の間の権限を動的に管理するシステム。最初のソフトウェアプロダクト。

最小構成

AgentCard（AIエージェント登録）、Composite Riskの8次元スコアリング、Autonomy Mode（A0–A5）自動判定、人間介入ポイントの設計UI、基本ダッシュボード

対応次元

D4D12

購買者・予算枠

CIO / CTO / CDO → IT・セキュリティ予算

→

Decision Ledgerへのデータ蓄積が始まる。Trust Scoreの初期値が形成される。

Phase 2 — 9〜18ヶ月
Integration

判断の記録・統治

権限配分エンジンに判断の記録・監査・改善機能を統合。事実上のSystem of Governanceが成立。

追加構成

Decision Ledger（全重要判断の記録）、Authority basisとaccountable ownerの紐付け、事前検証ゲート（判断実行前のpolicy / risk照合）、監査トレイル自動生成、判断パターン分析・レポート

対応次元

D4D8D10D11D12D13

購買者・予算枠

COO / CFO → GRC・コンプライアンス予算。監査法人連携が視野に入る。

→

判断データの蓄積が加速。複数企業のベンチマークが可能になり始める。ネットワーク効果の萌芽。

Phase 3 — 18〜30ヶ月
Internal Infra

Organizational World Model

蓄積されたデータを基盤に、組織の世界モデルを内部コンポーネントとして構築。外販はしない。

構成

既存SaaS群からの状態統合。6面構造の段階的実装（まずDecision PlaneとGovernance Planeから）。組織状態のリアルタイム可視化。

対応次元

D9D10

→

Phase 4（組織運営エンジン）の基盤が整う。判断の統治が「状況の理解」に裏打ちされるようになる。

Phase 4 — 30ヶ月〜
Evolution

組織運営エンジン

World Modelの上に業務ループ（Sense → Evolve）を載せ、「組織のOS」へ進化する。

構成

Constitution Load → Sense → Understand → Deliberate → Gate → Act → Observe → Reflect → Evolve の8ステップループ。Autonomy Modeに基づく動的な人間介入。Human-AI Collaboration機能の統合。

対応次元

D1D2D3D4D5D6D7D8D9D10D11D12D13

→

13次元の完全なソフトウェア実装。組織OS設計ツールは運営エンジンの設定UIとして自然発生する。

戦略的取捨

諦めるもの

組織OS設計ツールの独立化

Phase 4で運営エンジンの設定UIとして自然に生まれる。コンサルで同じことを人力でやれば十分。

世界モデル基盤の外販

内部インフラとして構築する。データ基盤市場で戦う必要はない。

最初から全13次元のカバー

Phase 1はD4とD12の一部だけでよい。完全性は段階的に達成する。

絶対に諦めないもの

Decision Ledger

全Phaseを貫くセンターピン。データの蓄積源、ネットワーク効果の基盤、13次元の実装起点。

13次元×5要素の知的体系

プロダクトの設計言語であり、競合が模倣できない構造的優位。

コンサルとソフトウェアのフライホイール

コンサルの経験がソフトウェアを改善し、ソフトウェアの利用データがコンサルの質を上げる循環。
